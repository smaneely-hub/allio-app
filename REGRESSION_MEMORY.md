# REGRESSION_MEMORY.md

## Repeat pain points to avoid

1. Do not put model-style intelligence into the frontend unless explicitly temporary.
2. Do not split one conceptual workflow into multiple screens/tabs without strong reason.
3. Do not rely on build success alone; verify live behavior.
4. Do not let cached local state override intentional edit-mode behavior.
5. Do not ask users for redundant fields if they can be inferred.
6. Do not add extra UI to compensate for unclear product thinking.
7. Do not patch around the wrong concept; replace the concept.
8. Do not treat working code as success if it violates the product direction.
